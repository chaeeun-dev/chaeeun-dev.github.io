## [C++]

### 이전 수업 내용 복습

- private으로 멤버 변수를 생성하면, 외부에서 변수에 직접 접근할 수 없어 getter와 setter를 통해 접근해야 한다.
- 부모 클래스이면 소멸자를 ‘virtual’로 선언해야 한다. 자식의 소멸자까지 올바르게 호출되기 위해서! 만약 virtual로 선언되지 않으면, 부모 클래스의 포인터를 통해 자식 클래스 객체를 삭제할 때 자식 클래스의 소멸자가 호출되지 않아 메모리 leak의 문제가 발생할 수 있다.
- 오버라이딩은 상속 받은 자식 클래스 에서 부모 클래스의 함수를 재정의 하는 것이다. override 키워드를 붙이는 이유는 부모 클래스의 함수를 오버라이딩 하고 있음을 명확하게 나타내기 때문이다. 필수는 아니다!
- 함수에서 인자를 받을 때 if ( item == nullptr) 이렇게 예외 처리를 꼭 하자. 꼼꼼하게 체크하는 습관을 들이자.
- 전방 선언은 나중에 등장할 클래스를 예고하는 것이다. class B를 정의하고 있는데, class A를 사용해야 하는 경우, 위에 class A; 이렇게 적고, B의 cpp 파일에 헤더를 include한다. 헤더에 헤더를 include하는 습관은 꼬일 수 있어서 위험하니 전방 선언을 사용하자!
- 얕은 복사를 할 때 객체가 포인터나 참조를 포함하고 있다면, 두 객체가 동일한 주소를 가리키게 되어 한 객체의 변경이 다른 객체에 영향을 미친다. 깊은 복사는 포인터가 가리키는 데이터도 복사하기 때문에 두 객체는 독립적으로 존재한다.  - default 복사 생성자는 얕은 복사 생성자를 수행한다. 만약 클래스가 동적 메모리를 사용하는 멤버 변수를 가지고 있다면, 기본적으로 제공되는 복사 생성자를 사용하지 말고, 직접 복사 생성자를 정의해서 “깊은 복사”가 되도록 만들어야 한다.

- 재귀함수 
재귀함수를 잘못 사용하면 스택 프레임을 반환하지 못해 stackoverflow가 발생할 수 있어서 주의해서 사용하자

유클리드 호제법 재귀함수로 구현하기

```cpp
int GCD(int a, int b)
{
		if (b == 0)
			return a;
			
		return GCD(b, a % b);
}
```

재귀함수 처음에 접했을 때 이해가 진짜 안 됐는데, 지금도 어렵다... 코딩 시험에서도 기초적인 부분으로 나온다고 하니까 자주 연습하자
